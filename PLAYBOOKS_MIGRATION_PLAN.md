# Playbooks Migration Plan

## Why add `Playbooks`

The new site already has three strong desks:

- `Daily Brief`
- `Tool Reviews`
- `Case Studies`

What it still lacks is the operating layer between information and execution.

`Playbooks` should become the fourth desk that answers:

- what stack to use for a real task
- in what order to use it
- which old AIandBusiness material should be rewritten into evergreen guides

This keeps the new site focused on the theme:

> 写给想把 AI 生意做成的人。一份单人公司指南。

## Recommended information architecture

### Top-level site

- `Daily Brief`: what changed
- `Tool Reviews`: what is worth using
- `Case Studies`: what is working in practice
- `Playbooks`: how to put the stack together

### What a playbook should contain

Each playbook page should have:

- a concrete business task
- a recommended stack
- a working sequence
- links to relevant reviews and cases
- source material migrated from the legacy docs site

## Legacy docs strategy

The old docs site should be retained, but repositioned.

### Recommended role for `docs.aiandbusiness.com`

Use it as:

- `Library`
- `Archive`
- `Reference`

Do not use it as the main editorial front door anymore.

### Keep on the docs site

- AI basics
- prompt libraries
- policy / ethics / regulation
- robotics / autonomous driving / AI hardware
- broader encyclopedia-like reference pages

### Migrate or rewrite into the new site

- pages that can become guides
- pages that can become stacks
- pages that can become workflows
- pages that are strongly relevant to solo builders

## Execution principle

Do not copy legacy pages 1:1.

Instead:

1. Keep the old site as a source library.
2. Rewrite the strongest pages into editorial playbooks.
3. Link each playbook to tool reviews and case studies.
4. Let the docs site remain useful for long-tail and reference traffic.
